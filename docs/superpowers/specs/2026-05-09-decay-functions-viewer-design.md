# 衰减函数可视化对比页面 — 设计文档

**日期：** 2026-05-09
**状态：** 已通过用户评审
**目标产物：** `docs/index.html`（单文件，零构建）

## 背景

`PID` 库提供了 5 个衰减/激活函数（`PID.c` 第 245–304 行），用于让 PID 参数在误差衰减过程中按预设曲线变化。这些函数形状相近但细节差异明显，开源使用者很难只看公式就直观感受它们的区别。本页面让访问者打开浏览器即可叠加对比、滑动调参，降低选型门槛。

## 范围内函数

按 `PID.c` 注释里的"理论公式"渲染（不直接照搬当前代码，因为存在已知 bug）：

| 函数 | 公式 | 参数 | 默认值 | 滑条范围 |
|---|---|---|---|---|
| `dsigmoidn(z, a)` | $y = 1 - \dfrac{4e^{-az}}{(1+e^{-az})^2} = \tanh^2(az/2)$ | a | 1.0 | [0.1, 5.0] |
| `sigmoidabsx(z, a, b)` | $y = \dfrac{1}{1+b\,e^{-a\|z\|}}$ | a, b | a=1, b=1 | a∈[0.1,5], b∈[0.1,5] |
| `tanhabsx(z, a)` | $y = \tanh(a\|z\|)$ | a | 1.0 | [0.1, 5.0] |
| `px1(z, a)` | $y = 1 - \dfrac{1}{\sqrt{a\|z\|+1}}$ | a | 1.0 | [0.1, 5.0] |
| `obliquestepfun(z, x)` | $y = \min(\|z\|/x, 1)$ | x | 1.0 | [0.1, 5.0] |

### 已知 C 代码 bug（不在本次修复范围）

`sigmoidabsx` 的 C 实现 `return 1.0f / (b * exp(-a * z));` 与注释里的公式 `1/(1+b·e^(-a|z|))` 不一致，缺少 `1 +`，且未取 `|z|`。本页面按**注释里的正确公式**渲染。修复 C 代码本身需要单独的 Issue/PR，不在本设计内。在 `docs/index.html` 上以脚注或 README 中注明此差异。

## 技术栈

- **Chart.js v4**（折线绘图，~70KB gzip）— 通过 jsDelivr CDN 引入
- **KaTeX**（公式渲染，~150KB gzip）— 通过 jsDelivr CDN 引入；用 `auto-render` 扩展自动渲染 `$...$` 与 `$$...$$`
- **纯 HTML + CSS + 原生 JS**，单文件 `docs/index.html`
- 不引入任何打包器、框架、TypeScript

## 交互细节

- **复选框切换**：勾选/取消即时增删图表数据集，并显示/隐藏对应参数块
- **滑条拖动**：使用 `input` 事件实时重算（5 个函数 × 200 点 = 1000 次/帧的简单计算，无性能问题），用 `requestAnimationFrame` 节流避免重复渲染
- **数值显示**：滑条右侧显示当前数值，保留 2 位小数
- **重置按钮**：恢复全部参数到默认值，并保持当前的勾选状态
- **默认勾选**：进入页面时 `dsigmoidn` 与 `tanhabsx` 默认勾选（让用户立即看到内容），其他默认未勾选
- **采样**：固定 200 个等间距点；`obliquestepfun` 在 `|z|=x` 处不刻意加密（用户能从参数变化中观察到）

## 颜色方案

5 条曲线使用 ColorBrewer 的 `Set1` 调色板前 5 色（区分度高，色盲友好）：

| 函数 | 颜色 |
|---|---|
| `dsigmoidn` | `#e41a1c`（红） |
| `sigmoidabsx` | `#377eb8`（蓝） |
| `tanhabsx` | `#4daf4a`（绿） |
| `px1` | `#984ea3`（紫） |
| `obliquestepfun` | `#ff7f00`（橙） |

整体页面使用浅色主题（白底深字），符合阅读习惯，不做暗色切换。

## 不做（YAGNI）

- ❌ 暗色主题切换
- ❌ 导出图片 / CSV
- ❌ x 范围用户自定义
- ❌ 自定义函数表达式输入
- ❌ 函数数学性质标注（极值、导数等）
- ❌ 多语言切换（先中文为主，关键名词保留英文）

## 验证标准

实现完成后需满足：

1. 用 `python -m http.server` 或直接用 `file://` 在 Chrome/Edge/Firefox 打开 `docs/index.html`，无控制台报错
2. 5 个复选框任意组合勾选都能正确叠加曲线
3. 每个滑条拖动时图形实时更新（< 16ms 一帧）
4. KaTeX 公式正确渲染，分式分层显示
5. 窗口缩到 < 900px 宽时布局自动堆叠，不溢出
6. 离线（断网后）无法工作是预期的，因为依赖 CDN — 不需要 fallback
